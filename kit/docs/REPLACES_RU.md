# Справочник по замене плейсхолдеров

Этот документ представляет собой подробное руководство по замене конфигурационных токенов `<PLACEHOLDER>` в `CLAUDE.md`, командах (skills) и агентах при импорте **Universal Agent Kit** в ваш репозиторий.

---

## Конфигурационные плейсхолдеры

Эти токены должны быть заменены один раз при начальной настройке/слиянии комплекта с вашим проектом.

| Токен плейсхолдера | Описание | Frontend / Node.js | Backend (Go / Python) | Mobile (Kotlin / Swift) | Systems (Rust / C++) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `<PROJECT_NAME>` | Имя вашего репозитория/проекта. | `my-web-app` | `user-service` | `FastMediaSorter` | `game-engine` |
| `<CHAT_LANGUAGE>` | Язык общения агента с вами. | `Russian` / `English` | `English` | `Russian` | `English` |
| `<INDEX_DOC>` | Начальный документ / карта проекта. | `README.md` | `docs/ARCH.md` | `README.md` | `README.md` |
| `<PLAN_DIR>` | Папка со спецификациями и планами. | `PLAN` | `docs/plans` | `PLAN` | `PLAN` |
| `<SCRATCH_DIR>` | Игнорируемая git папка для темп-файлов.| `tmp` | `.scratch` | `build/tmp` | `target/tmp` |
| `<ID>` | Формат / паттерн ID тикета. | `TNNNN` (e.g. `T0042`) | `JIRA-NNN` | `FMS-NNN` | `TNNNN` |
| `<LOGGER>` | Имя вызова/фасада логгера. | `logger.info` | `logging.info` / `log.Printf` | `Log.d` / `os_log` | `log::info!` |
| `<SRC_ROOT>` | Главная папка с исходным кодом. | `src` | `src` / `.` (корень) | `app/src/main/java` | `src` |
| `<ARCH_LAYERS>` | Правила направления зависимостей. | `page -> widget -> api` | `handler -> service -> repo` | `UI -> VM -> Repo` | `mod -> impl -> core` |
| `<BUILD_CMD>` | Команда сборки проекта. | `npm run build` | `go build ./...` | `./gradlew assemble` | `cargo build` |
| `<TEST_CMD>` | Команда запуска тестов. | `npm test` | `pytest` / `go test ./...` | `./gradlew test` | `cargo test` |
| `<LINT_CMD>` | Команда запуска статического анализа. | `npm run lint` | `flake8` / `golangci-lint run` | `./gradlew lint` | `cargo clippy` |
| `<RUN_CMD>` | Команда запуска dev-окружения. | `npm run dev` | `python main.py` | `./gradlew installDebug` | `cargo run` |
| `<MAX_LOC>` | Лимит строк в файле до рефакторинга. | `500` | `400` | `600` | `500` |
| `<READONLY_ZONES>`| Пути, которые запрещено менять агенту. | `dist, node_modules` | `vendor, .venv` | `build` | `target` |

## Подробное описание плейсхолдеров

### `<PROJECT_NAME>`
Имя проекта. Используется в заголовках документов и логах для обозначения контекста работы агента.
* *Пример:* `universal-agent-kit`

### `<CHAT_LANGUAGE>`
Язык общения между человеком и ИИ-ассистентом. Обратите внимание, что код, имена символов, коммиты и комментарии в коде рекомендуется оставлять на английском языке для единообразия.
* *Пример:* `Russian` или `English`

### `<INDEX_DOC>`
Файл, с которого агент начинает изучение структуры репозитория. Должен содержать архитектурные детали верхнего уровня, инструкции по запуску или ссылки на другие доки.
* *Пример:* `README.md` или `docs/ARCHITECTURE.md`

### `<PLAN_DIR>`
Каталог для хранения спецификаций и планов работы над тикетами. Разделение *что* делать от *как* делать - ключевая часть методологии.
* *Пример:* `PLAN/`

### `<SCRATCH_DIR>`
Папка для временных файлов, логов тестов или резервных копий перед редактированием. **Должна быть добавлена в `.gitignore`**.
* *Пример:* `tmp/` или `.scratch/`

### `<ID>`
Префикс или шаблон идентификатора тикета. Все файлы планов и временные логи привязаны к ID (например, `<PLAN_DIR>/<ID>_<slug>.md`).
* *Пример:* `T0042` или `PROJ-101`

### `<LOGGER>`
Инструмент логирования проекта. Правила чистого кода (anti-slop) запрещают использовать `print` или `console.log` в релизном коде. Все отладочные логи должны идти через этот фасад.
* *Пример:* `console.log` (для простых JS-приложений), `logger.debug` (winston/logback) или `Log.d` (Android).

### `<SRC_ROOT>`
Папка, где находится логика приложения. Ограничивает область поиска кода для агента, отсекая тесты, сборки и конфиги.
* *Пример:* `src/` или `app/src/main/`

### `<ARCH_LAYERS>`
Определяет слои архитектуры и разрешенное направление зависимостей. Например, `UI -> VM -> Repo` означает, что слой UI может импортировать ViewModel, но ViewModel не может импортировать UI.
* *Пример:* `controller -> service -> repository`

### `<BUILD_CMD>`, `<TEST_CMD>`, `<LINT_CMD>`, `<RUN_CMD>`
Консольные команды для сборки, тестирования, проверки линтером и запуска проекта. Ассистент запускает их, чтобы убедиться в работоспособности изменений.
* *Примеры:*
  * Сборка: `npm run build` | `go build ./...` | `mvn compile`
  * Тесты: `npm test` | `pytest` | `cargo test`
  * Линтер: `npm run lint` | `golangci-lint run` | `cargo clippy`
  * Запуск: `npm run dev` | `python main.py` | `cargo run`

### `<MAX_LOC>`
Максимальное количество строк в одном файле. Используется как сигнал для разделения кода. Если файл превышает лимит, агент должен вынести логику во вспомогательный класс.
* *Пример:* `500`

### `<READONLY_ZONES>`
Список путей через запятую, которые агенту строго запрещено изменять (например, внешние библиотеки, генерируемый код, файлы миграций).
* *Пример:* `node_modules/, vendor/, gen/`

## Временные токены шаблонов

Эти токены **не являются** частью конфигурации. Они заполняются динамически вами или агентом в процессе работы над конкретной задачей:

* `<slug>` - Краткое описание тикета на английском через дефис (например, `add-login-button`).
* `<NN>` / `<NNNN>` - Порядковый номер тикета (например, `01` или `0042`).
* `<TODO>` - Задача, которую предстоит сделать.
* `<symbol>` - Имя класса, функции, структуры или переменной.
* `<path>` - Путь к файлу или папке.
* `???` - Неопределенное решение или проверка.
